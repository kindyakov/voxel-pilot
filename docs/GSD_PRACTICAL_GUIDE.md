# GSD — Практическое Руководство

## Суть за 30 секунд

GSD — это **фреймворк для AI-агентов**, который превращает хаотичную разработку в управляемый процесс. Каждая команда — это **специальный агент** с чётким workflow.

**Главная идея:** Не давай AI "просто писать код". Заставь его сначала понять задачу, потом спланировать, потом сделать, потом проверить.

---

## Жизненный Цикл Фичи

```
Ты: "Хочу новую фичу"
    │
    ▼
/gsd-discuss-phase N    → AI задаёт вопросы, рождаются требования
    │
    ▼
/gsd-plan-phase N       → AI пишет пошаговый план (PLAN.md)
    │
    ▼
/gsd-execute-phase N    → AI делает по шагам, сам коммитит
    │
    ▼
/gsd-verify-work N      → AI проверяет что работает
    │
    ▼
/gsd-complete-milestone → AI закрывает этап, обновляет STATE.md
```

## КОГДА и ЧТО будет (по-честному)

### 🎯 `/gsd-discuss-phase N`

**Когда использовать:**
- У тебя есть идея фичи, но ты **не знаешь как её делать**
- Фича затрагивает несколько подсистем
- Хочешь чтобы AI нашёл подводные камни ДО написания кода

**Что произойдёт:**
1. AI загрузит PROJECT.md, ROADMAP.md, STATE.md
2. Просканирует код, найдёт что уже есть
3. **Начнёт задавать тебе вопросы** через интерактивное меню
4. Вы вместе обсуждаете серые зоны (как обрабатывать ошибки, какие API использовать)
5. AI создаст файл `N-CONTEXT.md` с **принятыми решениями**

**Результат:**
```
.planning/phases/
└── 02-CONTEXT.md     ← Решения: "используем X, не используем Y, Z обрабатываем так-то"
```

**Чего НЕ будет:** Ни строчки кода. Только документ с решениями.

**Пример:**
```
/gsd-discuss-phase 2
Хочу чтобы бот умел автоматически фармить ресурсы

AI задаст вопросы:
- Какие именно ресурсы? (дерево, руда, еда?)
- Бот должен сам искать или ты укажешь координаты?
- Что делать если моб напал?
- Как определять что инвентарь полон?

После обсуждения создаст 02-CONTEXT.md
```

### 📋 `/gsd-plan-phase N`

**Когда использовать:**
- AFTER `/gsd-discuss-phase` (есть CONTEXT.md)
- ИЛИ когда ты **точно знаешь** что хочешь и можешь описать

**Что произойдёт:**
1. AI прочитает CONTEXT.md (если есть)
2. Если флаг `--research`, сначала изучит библиотеки/документацию
3. Создаст `PLAN.md` с атомарными шагами в XML-формате
4. Каждый шаг = маленькая задача, которую можно закоммитить

**Результат:**
```
.planning/phases/
├── 02-CONTEXT.md     ← (если было обсуждение)
└── 02-PLAN.md        ← Пошаговый план:
                        <step id="1" task="Создать модуль pathfinder" />
                        <step id="2" task="Добавить обработку препятствий" />
                        <step id="3" task="..." />
```

**Чего НЕ будет:** Ни строчки кода. Только план.

**Пример:**
```
/gsd-plan-phase 2
На основе обсуждения в 02-CONTEXT.md

AI создаст 02-PLAN.md с 8-15 шагами
```

---

### ⚙️ `/gsd-execute-phase N`

**Когда использовать:**
- PLAN.md готов
- Ты готов что AI начнёт **менять код**

**Что произойдёт:**
1. AI прочитает PLAN.md
2. Разобьёт шаги на **волны** (параллельные группы)
3. Выполнит каждую волну:
   - Напишет код
   - Запустит тесты/type-check/build
   - **Сделает атомарный коммит**
   - Перейдёт к следующей волне
4. Обновит STATE.md

**Результат:**
```
git log:
✅ feat(phase-2): implement pathfinder module
✅ feat(phase-2): add obstacle handling
✅ feat(phase-2): integrate with HSM
...
```

**Чего НЕ будет:** AI не будет отступать от плана. Если нужно что-то другое — это в план надо было добавить.

**Пример:**
```
/gsd-execute-phase 2

AI выполнит все шаги из 02-PLAN.md
Каждый шаг = отдельный коммит
```

**Полезные флаги:**
```
/gsd-execute-phase 2 --wave 1      # Только первую волну (для контроля)
/gsd-execute-phase 2 --interactive # Последовательно, с паузами между шагами
/gsd-execute-phase 2 --gaps-only   # Только шаги для исправления багов
```

---

### ✅ `/gsd-verify-work N`

**Когда использовать:**
- После `/gsd-execute-phase N`
- Перед тем как закрыть этап

**Что произойдёт:**
1. AI прочитает PLAN.md и критерии успеха
2. Запустит тесты, type-check, build
3. Проверит что каждый критерий выполнен
4. Создаст `VERIFICATION.md` с отчётом

**Результат:**
```
.planning/phases/
└── 02-VERIFICATION.md  ← Отчёт:
                          ✅ Все тесты проходят
                          ✅ npm run type-check ок
                          ✅ Критерий 1: pathfinder работает
                          ❌ Критерий 2: нет обработки препятствий
```

**Если что-то не ок:** AI создаст fix-планы, ты можешь запустить `/gsd-execute-phase N --gaps-only`

---

### 🏁 `/gsd-complete-milestone N`

**Когда использовать:**
- Фаза выполнена и проверена
- Хочешь зафиксировать прогресс

**Что произойдёт:**
1. Обновит STATE.md (progress: X%)
2. Обновит ROADMAP.md (отметит фазу [x])
3. Зафиксирует принятые решения

**Результат:**
```
STATE.md:
  progress:
    completed_phases: 2
    percent: 100

ROADMAP.md:
  - [x] Phase 2: Auto-farming system
```

## Быстрые Команды (когда НЕ нужен план)

### 🔧 `/gsd-quick`

**Когда:**
- Багфикс (15 минут работы)
- Добавить логирование
- Поправить конфиг
- Одна функция

**Что произойдёт:**
1. AI поймёт задачу
2. Напишет код
3. Закоммитит
4. Обновит STATE.md в секции "Quick Tasks"

**Результат:**
```
.planning/quick/
└── task-2026-04-14.md  ← Что сделал

git log:
✅ fix: add logging to connectToServer
```

**Пример:**
```
/gsd-quick
Добавь таймаут 30 секунд к подключению к серверу

AI найдёт connectToServer, добавит timeout, закоммитит
```

---

### 🚀 `/gsd-fast`

**Когда:**
- Прототип "лишь бы работало"
- PoC для демонстрации
- Speedrun хакатон

**Предупреждение:** Будет грязный код, техдолг, no tests. Потом придётся переделывать.

**Пример:**
```
/gsd-fast
Набросай endpoint /health чтобы возвращал {"status": "ok"}

AI создаёт роут без валидации, без тестов, без обработки ошибок
```

## Частые Ошибки

| Ошибка | Почему плохо | Как правильно |
|--------|--------------|---------------|
| `/gsd-execute` без плана | AI делает как понял → переделки | Сначала discuss+plan |
| Пропускать `/gsd-verify` | Не знаешь что сломалось | Всегда проверяй |
| Править код во время execute | Ломает атомарность коммитов | Добавь в план или `/gsd-quick` |
| Забывать `/gsd-complete-milestone` | STATE.md враньё | Закрывай этапы |
| Использовать `/gsd-fast` для продакшена | Техдолг | Только для прототипов |

## Полезные Флаги

| Команда | Флаг | Что делает |
|---------|------|------------|
| `/gsd-plan-phase` | `--research` | Изучит библиотеки/доки перед планированием |
| `/gsd-plan-phase` | `--skip-research` | Пропустить исследование |
| `/gsd-execute-phase` | `--wave N` | Только волну N (контроль) |
| `/gsd-execute-phase` | `--interactive` | Пошагово с паузами (парное программирование) |
| `/gsd-execute-phase` | `--gaps-only` | Только исправления (после verify) |
| `/gsd-quick` | `--full` | Полная проверка (discuss+research+verify) |
| `/gsd-quick` | `--research` | Сначала исследовать подход |
| `/gsd-discuss-phase` | `--auto` | AI сам выберет варианты (без вопросов) |
| `/gsd-discuss-phase` | `--chain` | Discuss → Plan → Execute сразу |

## Артефакты (файлы которые создаются)

| Файл | Где лежит | Кто создаёт | Что внутри |
|------|-----------|-------------|------------|
| `PROJECT.md` | `.planning/` | `/gsd-new-project` | Описание проекта, ценности |
| `ROADMAP.md` | `.planning/` | `/gsd-new-project` | Список всех фаз |
| `STATE.md` | `.planning/` | Все команды | Текущий прогресс, решения |
| `N-CONTEXT.md` | `.planning/phases/` | `/gsd-discuss-phase N` | Принятые решения |
| `N-PLAN.md` | `.planning/phases/` | `/gsd-plan-phase N` | Пошаговый план |
| `N-VERIFICATION.md` | `.planning/phases/` | `/gsd-verify-work N` | Отчёт проверки |
| `quick/task-*.md` | `.planning/quick/` | `/gsd-quick` | Что сделала быстрая правка |

## Когда Что Использовать (Дерево Решений)

```
У тебя есть задача. Смотри на объём:

Меньше 15 минут кода?
  └─ /gsd-quick "текст задачи"

Прототип / PoC / проверить идею?
  └─ /gsd-fast "текст задачи"

Больше 15 минут?
  │
  Знаешь КАК делать? (конкретные шаги)
  │  └─ /gsd-plan-phase N "опиши что знаешь"
  │     └─ /gsd-execute-phase N
  │
  НЕ знаешь КАК делать? (серые зоны, варианты)
     └─ /gsd-discuss-phase N "опиши идею"
        └─ /gsd-plan-phase N
           └─ /gsd-execute-phase N
              └─ /gsd-verify-work N
                 └─ /gsd-complete-milestone
```

## Главное Что Нужно Понимать

1. **GSD — это не про тебя, это про AI**  
   Команды заставляют AI следовать процессу, а не импровизировать

2. **Каждая команда = отдельный агент**  
   `discuss` ≠ `plan` ≠ `execute`. У каждого своя роль.

3. **Артефакты — это контракт**  
   CONTEXT.md, PLAN.md, VERIFICATION.md — это документы которые AI обязан соблюдать

4. **Можно вмешиваться**  
   После `/gsd-discuss` ты можешь отредактировать CONTEXT.md  
   После `/gsd-plan` ты можешь отредактировать PLAN.md  
   AI будет следовать отредактированной версии

5. **Состояние сохраняется**  
   STATE.md — это память. AI помнит что было сделано между сессиями
